Unit 1 · Introduction
Basic Computer Organization
A computer system is built from functional units connected by buses. Understanding how data flows between these units is the foundation of COA.
💡
WHY THIS MATTERS
Foundation of all computer systems

Understanding computer organization is like understanding how a car works. Each component has a specific job:

  • CPU is the brain - Makes all decisions and calculations
  • Memory is the workspace - Stores data temporarily while working
  • I/O devices are the interface - Connect you to the computer
  • Buses are highways - Data travels through these paths
🎯 Exam Tip: Always label buses (data/address/control) correctly in diagrams!
🏗 Complete Block Diagram
🧠 Functional Units
ALU — Arithmetic Logic Unit
Performs all calculations (+, −, ×, ÷) and logical operations (AND, OR, XOR, NOT, shifts). Gets operands from registers.
CU — Control Unit
Fetches instructions, decodes them, and sends control signals to all other units. The "director" of the CPU.
Registers
Ultra-fast storage inside CPU. PC (program counter), IR (instruction register), MAR, MDR, ACC, R0–R7.
Memory
Stores programs and data. CPU sends address via MAR, data comes back via MDR.

Unit 1 · Number Systems
Number Representation
Computers store numbers in binary. Understanding signed/unsigned representation, fixed-point, and integer formats is crucial for all arithmetic operations.
📖 STEP-BY-STEP: Binary ↔ Decimal Conversion
1️⃣ Binary to Decimal
Example: Convert 1101₂ to decimal
Step 1: Write powers: 2³ 2² 2¹ 2⁰
Step 2: Place bits: 1 1 0 1
Step 3: Multiply: 8 + 4 + 0 + 1
Result = 13₁₀
✓ FORMULA:
Multiply each bit by 2ⁿ where n is position from right (starting at 0)
2️⃣ Decimal to Binary
Example: Convert 13₁₀ to binary
13 ÷ 2 = 6 remainder 1 ←LSB
6 ÷ 2 = 3 remainder 0
3 ÷ 2 = 1 remainder 1
1 ÷ 2 = 0 remainder 1 ←MSB
Read ↑ upward = 1101₂
✓ METHOD:
Divide by 2 repeatedly, note remainders, read BOTTOM to TOP
⚠️ COMMON MISTAKES:
  • Forgetting to read remainders from BOTTOM to TOP in decimal→binary
  • Starting position count from 1 instead of 0
  • Mixing up MSB (leftmost) and LSB (rightmost)

1's & 2's Complement + Overflow
2's complement is the universal standard for signed integers. It allows subtraction to be performed using addition, simplifying hardware design.
\n
\n
🤔 WHY 2's COMPLEMENT? (Understand the REASON)
\n
\n

Problem: How do we represent negative numbers in binary?

\n \n
\n
❌ Attempt 1: Sign-Magnitude (MSB = sign)
\n
\n +5 = 0101, −5 = 1101
\n ✗ TWO zeros: 0000 (+0) and 1000 (−0)
\n ✗ Addition circuit is COMPLEX (need to check signs first)\n
\n
\n\n
\n
⚠️ Attempt 2: 1's Complement (flip all bits)
\n
\n +5 = 0101, −5 = 1010
\n ✗ STILL two zeros: 0000 and 1111
\n ⚠️ Need end-around carry in addition\n
\n
\n\n
\n
✓ Solution: 2's Complement (1's comp + 1)
\n
\n +5 = 0101, −5 = 1011
\n ✓ ONLY ONE zero: 0000
\n ✓ Addition/Subtraction use SAME circuit!
\n ✓ Just discard final carry – no special handling\n
\n
\n\n
\n 💡 Key Insight: In 2's complement, A − B = A + (−B) = A + 2's_complement(B)
\n This means subtraction is just addition with negated second operand!\n

⚠️ Overflow Detection (EXAM IMPORTANT)
Overflow occurs when the result cannot be represented in n bits.
Rule 1: V = Carryin ⊕ Carryout

Rule 2: Two positives → negative result
Two negatives → positive result
• No Overflow: Cin = Cout of MSB
• Overflow: Cin ≠ Cout of MSB

IEEE-754 Floating Point
IEEE 754 is the international standard for floating-point arithmetic. It defines 32-bit (single) and 64-bit (double) precision formats used in ALL modern processors.

📝 Step-by-Step Conversion (13.375 Example)
Step 1: Sign Bit
13.375 is positive → S = 0
Step 2: Convert to Binary
13 = 1101₂
0.375 × 2 = 0.75 → 0
0.75 × 2 = 1.5 → 1
0.5 × 2 = 1.0 → 1
∴ 0.375 = 0.011₂
→ 13.375 = 1101.011₂

Step 4: Compute Exponent

Unit 2 · Combinational Circuits
Half Adder & Full Adder
Adders are the most fundamental arithmetic circuits. Built from basic logic gates (XOR, AND), they form the basis of all ALU arithmetic operations.
① Half Adder — 2 inputs, no carry-in
Sum = A ⊕ B (XOR)
Carry = A · B (AND)
ABSumCarry
0000
0110
1010
1101
Limitation: Cannot handle carry-in, so can only be used for the least significant bit in a multi-bit adder.
② Full Adder — 3 inputs (A, B, Cin)
Sum = A ⊕ B ⊕ Cin
Cout = (A·B) + (B·Cin) + (A·Cin)
ABCinSumCout
Key insight: A Full Adder = 2 Half Adders + 1 OR gate. It can handle a carry input, making it suitable for all bit positions in a multi-bit adder.
